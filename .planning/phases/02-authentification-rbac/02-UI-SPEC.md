## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label |
| sm | 8px | Compact internal element spacing (input padding, tight stacks) |
| md | 16px | Default element spacing (form field gaps, card padding) |
| lg | 24px | Section padding (onboarding step container) |
| xl | 32px | Layout gaps (space between wizard header and content) |
| 2xl | 48px | Major section breaks (space between form groups in onboarding) |
| 3xl | 64px | Page-level spacing (login page vertical centering offset) |

Exceptions:
- Touch targets for buttons: minimum 44px height (WCAG 2.5.5)
- Step indicator circles: 32px × 32px (8px × 4 scale)

## Color

All values reference Tailwind CSS v3 palette tokens. Dark mode via shadcn CSS variable theming.

| Role | Tailwind Token | Hex (light) | Usage |
|------|----------------|-------------|-------|
| Dominant (60%) | `white` / `zinc-50` | #ffffff / #fafafa | Page background, form surfaces, card backgrounds |
| Secondary (30%) | `zinc-100` / `zinc-200` | #f4f4f5 / #e4e4e7 | Left panel background, input backgrounds, dividers |
| Accent (10%) | `indigo-600` | #4f46e5 | Reserved — see list below |
| Destructive | `red-600` | #dc2626 | Destructive actions only (no Phase 2 destructive actions identified) |

**Accent (`indigo-600`) reserved exclusively for:**
1. Primary CTA button (send magic link, advance wizard step)
2. Active step circle in the onboarding step indicator
3. Focus ring on form inputs (Tailwind `ring-indigo-600`)
4. Right-panel gradient: `from-indigo-500/20 to-indigo-700/40` (login page decorative background)
5. Link text in confirmation screen ("renvoyer le lien")

Accent is NOT used for: icons, labels, secondary buttons, borders, body text, headings, card borders.

## Screen-Level Contracts

### Screen 1 — Login Page (`/login`)

**Layout:** Split-screen, horizontal 50/50 on desktop. Single column on mobile (right panel hidden below `md` breakpoint).

**Left panel:**
- Background: `white`
- Vertical centering via flexbox
- Content max-width: 360px, centered in the panel
- Logo: CabinetPilot wordmark, top-left anchor, 24px height
- Form title: "Connexion" — 20px DM Sans semibold, `zinc-900`
- Subtitle: "Pas de mot de passe — connexion sécurisée par lien e-mail" — 14px DM Sans regular, `zinc-500`
- Email input: shadcn `<Input>` with label "Adresse e-mail", placeholder "vous@cabinet.fr"
- Primary CTA: shadcn `<Button>` variant `default` — label "Recevoir mon lien de connexion", full width, `indigo-600` background
- Input validation: inline error below the field — red-600 text, 14px — "Adresse e-mail invalide" or "Trop de tentatives, réessayez dans quelques minutes"

**Right panel:**
- Background: gradient `from-indigo-500/20 to-indigo-700/40`
- Centered text block (max-width 320px):
  - Headline: 28px Cal Sans, `indigo-900` (or white if contrast sufficient)
  - 2–3 lines value proposition copy (see Copywriting section)
- Decorative only — no interactive elements

**After magic link sent (in-page state change — no navigation):**
- Left panel content replaces form with confirmation block:
  - Icon: envelope (Lucide `Mail`, 32px, `indigo-600`)
  - Heading: "Vérifiez votre boîte mail" — 20px DM Sans semibold
  - Body: "Un lien de connexion a été envoyé à [email@address]. Le lien expire dans 1 heure." — 16px DM Sans regular, `zinc-600`
  - Secondary action: "Renvoyer le lien" — text link, `indigo-600`, 14px — rate-limited on the backend
  - Back link: "← Modifier l'adresse e-mail" — 14px `zinc-500`

### Screen 2 — Onboarding Wizard (`/onboarding`)

**Layout:** Single centered column. Max-width 520px container, vertically centered in viewport. Background: `zinc-50`.

**Step indicator (top of page, above wizard card):**
- 5 numbered circles, horizontally spaced
- Active step: 32px circle, `indigo-600` fill, white number, DM Sans semibold 14px
- Completed step: 32px circle, `indigo-100` fill, `indigo-600` checkmark icon (Lucide `Check`, 16px)
- Upcoming step: 32px circle, `zinc-200` fill, `zinc-400` number
- Connecting line between circles: 1px `zinc-200`
- Step labels below each circle: 12px DM Sans regular, `zinc-400` (inactive) / `indigo-600` (active)

Step labels:
1. Profil
2. Groupe
3. Pennylane
4. Filtres
5. Sync

**Wizard card:**
- Background: `white`
- Border radius: `lg` (8px)
- Border: 1px `zinc-200`
- Box shadow: `shadow-sm`
- Padding: 32px

**Framer Motion transitions between steps:**
- Slide direction: advancing → slide left (exit: x: -60, opacity: 0), entering: x: 60 → 0
- Retreating → slide right (exit: x: 60, opacity: 0), entering: x: -60 → 0
- Duration: 250ms, ease: `easeInOut`
- Use `AnimatePresence` with `mode="wait"`

**Step 1 — Profil (obligatoire)**
- Section heading: "Votre profil" — 20px DM Sans semibold
- Fields: Prénom (required), Nom (required), Téléphone (optional), Photo de profil (optional, upload)
- Photo upload: shadcn-compatible file input, shows avatar preview after selection (circular, 64px)
- CTA: "Continuer" — `indigo-600`, full-width button
- No skip option on this step

**Step 2 — Groupe (obligatoire)**
- Section heading: "Votre cabinet"
- Fields: Nom du groupe (required) — e.g. "Cabinet Dupont & Associés"
- Helper text below input: "Ce nom sera visible par tous les membres de votre organisation"
- CTA: "Continuer"
- No skip option on this step

**Step 3 — Pennylane (passable)**
- Section heading: "Connexion Pennylane"
- Fields: Hôte Redshift, Port (default 5439), Nom de la base, Utilisateur, Mot de passe (password input)
- "Tester la connexion" button: shadcn `<Button>` variant `outline`, full-width, positioned below credentials fields
  - Loading state: Lucide `Loader2` spinner 16px, animated, label "Test en cours..."
  - Success state: Lucide `CheckCircle2` 16px `green-600`, label "Connexion réussie"
  - Error state: Lucide `XCircle` 16px `red-600`, label + inline error message below button
- CTA: "Continuer"
- Skip link: "Configurer plus tard" — 14px `zinc-400`, text link, no underline, positioned below CTA

**Step 4 — Filtres (passable)**
- Section heading: "Filtres de synchronisation"
- Checkboxes (shadcn `<Checkbox>`): Exclure LMNP, Exclure BNC, Exclure EI (codes 1000/1), Exclure dossiers "situation"
- Text input: "Codes juridiques exclus" — comma-separated, placeholder "ex: 5499, 5710"
- Date input: "Date de début d'exercice minimum" — shadcn `<Input>` type date
- CTA: "Continuer"
- Skip link: "Passer cette étape" — same styling as Step 3 skip

**Step 5 — Première sync (passable)**
- Section heading: "Première synchronisation"
- Description: 16px DM Sans regular — "Importez vos dossiers Pennylane pour démarrer." — `zinc-600`
- Primary CTA: "Synchroniser maintenant" — `indigo-600`, full-width
  - Loading state: Lucide `Loader2` spinner + "Synchronisation en cours..."
  - Success state: Summary card (see Copywriting section)
  - Error state: Inline error message (see Copywriting section)
- Skip link: "Passer et aller au dashboard" — 14px `zinc-400`
- After success OR skip: CTA changes to "Accéder au dashboard" — `indigo-600`

## Accessibility Requirements

- All form inputs must have associated `<label>` elements (shadcn Form components handle this)
- Error messages must be linked to inputs via `aria-describedby`
- Step indicator must have `aria-current="step"` on the active step
- Wizard card transitions must respect `prefers-reduced-motion` — use `opacity` only (no slide) when motion is reduced
- All buttons must have visible focus rings (ring-2 ring-indigo-600 ring-offset-2)
- Color contrast: body text `zinc-900` on `white` = 15.3:1 (AAA). Secondary text `zinc-500` on `white` = 4.6:1 (AA).
- Minimum touch target: 44px height on all buttons
